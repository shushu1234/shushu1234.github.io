            343-整数拆分(Integer Break)

            LiuYao 2018-07-24 数学<Math> 动态规划<Dynamic Programming>

            # 中文题目

            给定一个正整数 n，将其拆分为至少两个正整数的和，并使这些整数的乘积最大化。 返回你可以获得的最大乘积。

            示例 1:

            输入: 2
输出: 1
解释: 2 = 1 + 1, 1 × 1 = 1。

            示例 2:

            输入: 10
输出: 36
解释: 10 = 3 + 3 + 4, 3 × 3 × 4 = 36。

            说明: 你可以假设 n 不小于 2 且不大于 58。

            class Solution {
    private int[] memo;
        public  int integerBreak(int n) {
            memo=new int[n+1];
            Arrays.fill(memo,-1);
            return breakInteger(n);
        }

       private  int breakInteger(int n) {
            if (n==1){
                return 1;
            }
            if (memo[n]!=-1){
                return memo[n];
            }
            int res=-1;
           for (int i = 1; i <= n-1; i++) {
              res=Math.max(res,Math.max(i*(n-i),i*breakInteger(n-i)));
           }
           memo[n]=res;
           return res;
       }
}

            # 高赞题解

            # 解题思路：

            • 首先，任何一个数字 nnn，都可以被分为有限个小数字之和 n=n1+n2+...+nMn = n_1 + n_2 + ... + n_Mn=n1​+n2​+...+nM​。

              • 根据常理： 一般这 MMM 个数字的乘积要大于原数字 nnn。

            • 其次，所有数字nnn 都可以通过对一个因子 xxx 求整数部分 aaa（a = n // x） 和余数部分 bbb （b = n % x）；

              • 即得出数字 nnn 由 aaa 个 xxx 和 111 个 bbb 相加而成。

            • 问题转化： 是否有优先级最高的因子 xxx 存在？若有，我们就可以把问题转化为求 xa∗bx^a * bxa∗b 这个表达式的最大值。

              • 例如：2=1+12 = 1 + 12=1+1，1∗1<21 * 1 < 21∗1<2，因此 222 比 1+11+11+1 更优；

              • 例如：3=1+23 = 1 + 23=1+2，1∗2<31 * 2 < 31∗2<3，因此 333 比 111 和 222 更优；

              • 例如：4=2+24 = 2 + 24=2+2，2∗2=42 * 2 = 42∗2=4，因此可以认为 444 与 222 等价，因此见到 444 就拆分；

              • 例如：5=2+35 = 2 + 35=2+3；因为每个 555 都可以拆分为 2+32+32+3，而 2∗3=6>52 * 3 = 6 > 52∗3=6>5，因此见到 555 就拆分。

              • 例如：6=3+3=2+2+26 = 3 + 3 = 2 + 2 + 26=3+3=2+2+2；因为 3∗3>2∗2∗2>63 * 3 > 2 * 2 * 2 > 63∗3>2∗2∗2>6。因此见到 666 就拆分，并且 333 是比 222 更优的因子。

              • 易推出： 大数字都可以被拆分为多个小因子，以获取更大的乘积，只有 222 和 333 不需要拆分。

                n 拆分 乘积
                2 1+1 1 不拆分，2 比 1+1 更优
                3 1+2 2 不拆分，3 比 1+2 更优
                4 2+2 4 拆分，2 与 4 等价
                5 2+3 6 拆分
                6 3+3 9 拆分，3+3 比 2+2+2 更优
                7 2+2+3 12 拆分，但不能拆成 1+3+3

            • 观察以上枚举，我们可以列出以下贪心法则：

              • 第一优先级： 333；把数字 nnn 拆成尽可能多的 333 之和；
                • 特殊情况： 拆完后，如果余数是 111；则应把最后的 3+13 + 13+1 替换为 2+22 + 22+2，因为后者乘积更大；
              • 第二优先级： 222；留下的余数如果是 222，则保留，不再拆为 1+11+11+1。

            • 算法流程：

              1. 当 n<=3n <= 3n<=3 时，按照贪心规则应直接保留原数字，但由于题目要求必须拆分，因此必须拆出一个 111，即直接返回 n−1n - 1n−1；
              2. 求 nnn 除以 333 的整数部分 aaa 和余数部分 bbb；
              3. 当 b==0b == 0b==0 时，直接返回 3a3^a3a；
              4. 当 b==1b == 1b==1 时，要将一个 1+31 + 31+3 转换为 2+22+22+2，此时返回 3a−1∗23^{a-1} * 23a−1∗2；
              5. 当 b==2b == 2b==2 时，返回 3a∗b3^a * b3a∗b。

            # 复杂度分析：

            • 时间复杂度 O(1)O(1)O(1) ： 仅有求整、求余、次方运算。
              • 求整和求余运算：查阅资料，提到不超过机器数的整数可以看作是 O(1)O(1)O(1) ；
              • 次方运算：查阅资料，提到不超过机器数的整数可以看作是 O(1)O(1)O(1) 。
            • 空间复杂度 O(1)O(1)O(1) ： a 和 b 使用常数大小额外空间。

            class Solution:
    def integerBreak(self, n: int) -> int:
        if n <= 3: return n - 1
        a, b = n // 3, n % 3
        if b == 0: return pow(3, a)
        if b == 1: return pow(3, a - 1) * 4
        return pow(3, a) * 2

            class Solution {
    public int integerBreak(int n) {
        if(n <= 3) return n - 1;
        int a = n / 3, b = n % 3;
        if(b == 0) return (int)Math.pow(3, a);
        if(b == 1) return (int)Math.pow(3, a - 1) * 4;
        return (int)Math.pow(3, a) * 2;
    }
}
