            303-区域和检索 - 数组不可变(Range Sum Query - Immutable)

            LiuYao 2018-05-08 动态规划<Dynamic Programming>

            # 中文题目

            给定一个整数数组  nums，求出数组从索引 i 到 j  (i ≤ j) 范围内元素的总和，包含 i,  j 两点。

            示例：

            给定 nums = [-2, 0, 3, -5, 2, -1]，求和函数为 sumRange()

sumRange(0, 2) -> 1
sumRange(2, 5) -> -1
sumRange(0, 5) -> -3

            说明:

            1. 你可以假设数组不可变。
            2. 会多次调用 sumRange 方法。

            # 通过代码

            class NumArray {

    int[] nums;
        public NumArray(int[] nums) {
            for (int i = 1; i <nums.length ; i++) {
                nums[i]+=nums[i-1];
            }
            this.nums=nums;
        }

        public int sumRange(int i, int j) {
           if (i==0){
               return nums[j];
           }
           return nums[j]-nums[i-1];
        }
}

/**
 * Your NumArray object will be instantiated and called as such:
 * NumArray obj = new NumArray(nums);
 * int param_1 = obj.sumRange(i,j);
 */

            # 官方题解

            # 方法一：暴力法[超过时间限制]

            每次调用 sumrange 时，我们都使用for循环将索引 iii 到 jjj 之间的每个元素相加。

            private int[] data;

public NumArray(int[] nums) {
    data = nums;
}

public int sumRange(int i, int j) {
    int sum = 0;
    for (int k = i; k <= j; k++) {
        sum += data[k];
    }
    return sum;
}

            复杂度分析

            • 时间复杂度：每次查询的时间 O(n)O(n)O(n)，每个 sumrange 查询需要 O(n)O(n)O(n) 时间。
            • 空间复杂度：O(1)O(1)O(1)，请注意，data 是对 nums 的引用，不是它的副本。

            # 方法二：缓存

            假设 sumrange 被调用 1000次，其参数完全相同。我们怎么能加快速度？ 我们可以用额外的空间换取速度。通过预先计算所有的范围和可能性并将其结果存储在哈希表中，我们可以将查询加速到常量时间。

            private Map<Pair<Integer, Integer>, Integer> map = new HashMap<>();

public NumArray(int[] nums) {
    for (int i = 0; i < nums.length; i++) {
        int sum = 0;
        for (int j = i; j < nums.length; j++) {
            sum += nums[j];
            map.put(Pair.create(i, j), sum);
        }
    }
}

public int sumRange(int i, int j) {
    return map.get(Pair.create(i, j));
}

            # 方法三：缓存

            • 上面的方法需要很大的空间，我们可以优化它吗？
            • 假设我们预先计算了从数字 000 到 kkk 的累积和。我们可以用这个信息得出 sum(i，j)sum(i，j)sum(i，j) 吗？
            • 让我们将 sum[k]sum[k]sum[k] 定义为 nums[0⋯k−1]nums[0\cdots k-1]nums[0⋯k−1] 的累积和（包括这两个值）：

            sum[k]={∑i=0k−1nums[i],k>00,k=0sum[k] = \left\{ \begin{array}{rl} \sum_{i=0}^{k-1}nums[i] & , k > 0 \\ 0 &, k = 0 \end{array} \right. sum[k]={∑i=0k−1​nums[i]0​,k>0,k=0​

            • 现在，我们可以计算 sumrange 如下：

            sumrange（i，j）=sum[j+1]−sum[i]sumrange（i，j）=sum[j+1]-sum[i] sumrange（i，j）=sum[j+1]−sum[i]

            private int[] sum;

public NumArray(int[] nums) {
    sum = new int[nums.length + 1];
    for (int i = 0; i < nums.length; i++) {
        sum[i + 1] = sum[i] + nums[i];
    }
}

public int sumRange(int i, int j) {
    return sum[j + 1] - sum[i];
}
